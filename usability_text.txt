Executive Summary 
This report presents the findings from six usability studies conducted for GigTrack, a full-stack 
web application designed to help students and freelancers track their gig work, rate clients, 
monitor earnings, and set income goals. The studies followed a structured methodology aligned 
with course requirements, focusing on user-centric tasks based on defined user personas and 
stories. 
Key Findings: 
 Participants consistently understood the application's purpose within 2-3 minutes of 
exploration 
 All six participants successfully completed core tasks with minimal assistance 
 The health status feature for goals received universal positive feedback 
 Several usability improvements were identified, including enhanced filtering and reduced 
data entry redundancy 
1. Introduction 
1.1 Purpose of Study 
The purpose of this usability study was to evaluate the effectiveness and ease of use of 
GigTrack, identify pain points in the user experience, and gather actionable feedback for 
iterative improvement before the final project submission (Project 4). 1.2 Application Overview 
GigTrack is a personal side hustle tracker for students in the gig economy. It allows users to: 
 Log every gig with full details (client, type, date, hours, rate, earnings) 
 Rate clients and add notes for future reference 
 Track earnings across different gig types via a visual dashboard 
 Set monthly income goals and track progress with color-coded health status 
 Log payouts as received or pending against goals 
The application consists of two independent but complementary modules: 
 Gigs Module (Jinam): Gig logging, filtering, client ratings, earnings dashboard 
 Goals Module (Sanket): Income goals, payout tracking, health status, goal streaks

3. Tasks Performed 
Each participant completed multiple user-centric tasks designed to evaluate specific user stories. Task 1: Log a New Gig 
User Story: As a user, I want to log a gig with title, client name, gig type, date, hours worked, and 
rate so I have a full record of my work. 
Scenario Example (Caleb): "You just finished a 3-hour tutoring session with a student named 
Alex, and they paid you $60. Go ahead and log this gig." 
Success Criteria: 
 Participant finds the "+ Log a Gig" button without being told 
 Participant fills out all required fields correctly 
 Participant successfully saves the gig and sees it in the list 
Results: All 6 participants (100%) completed this task successfully in an average of 4.7 minutes. 
Task 2: Rate a Client and Add a Note 
User Story: As a user, I want to rate my client on a 1-5 scale and add a short note so I remember 
who was good to work with. 
Scenario Example (Taylor): "That delivery was frustrating — the restaurant took forever and the 
customer tipped poorly. Add a note and give it a low rating." 
Results: 5 of 6 participants (83%) completed this task successfully in an average of 5.2 minutes. 
Task 3: Check Earnings Dashboard / Filter by Gig Type 
User Story: As a user, I want to filter my gigs by type so I can find specific entries quickly and see 
which gig type earns the most. 
Scenario Example (Chet): "You want to compare your earnings from delivery work versus retail 
work. Show only your delivery gigs and tell me how much you made." 
Results: All 6 participants (100%) completed this task successfully in an average of 4.3 minutes. 
Task 4: Create a Goal and Log Payouts 
User Story: As a user, I want to create a monthly income goal and log payouts to track my 
progress toward hitting my target. 
Scenario Example (Scarlet): "You're saving for a $2000 vacation to Japan. Set up a monthly 
income goal of $500, then log a $150 payment you received." 
Results: All 5 participants tested (100%) completed this task successfully in an average of 7.6 
minutes. 4. Success Criteria and Measurements

4.2 Definition of Success 
 Completion: Did the participant finish the task? 
 Independence: Could they complete it without moderator assistance? 
 Time: Did they complete it within 5-7 minutes? 
 Accuracy: Was the data entered correctly and saved? 
 Confidence: Did the participant express confidence in their actions?

8. Conclusion 
The usability studies for GigTrack were highly successful in identifying both strengths and areas 
for improvement. 
What Worked Well: 
 Intuitive Forms: All participants found the data entry forms easy to understand and use 
 Health Status Indicator: The color-coded goal status was universally praised 
 Filter Functionality: The gig type filter was found and used correctly by all participants 
 Overall Value: Participants with actual gig experience expressed genuine interest in using 
the application Key Areas for Improvement: 
 Visibility: Rating fields and quick actions need to be more prominent 
 Filtering: Users expect rating-based filters in addition to type filters 
 Efficiency: Double-entry between gigs and payouts should be reduced 
 Comparison: Multi-month views would help users understand trends
